# AnimusForge Mod 一键编译脚本

## 使用方法

### 方式一：双击运行（推荐）

直接双击 `一键编译.bat` 即可自动完成编译和部署。

### 方式二：PowerShell

右键 `build.ps1` 选择"使用PowerShell运行"

## 配置说明

编译配置在 `build.ps1` 文件的开头：

```powershell
# ============ 配置区域 - 请修改这里的路径 ============
$BANNERLORD_ROOT = "C:\SteamLibrary\steamapps\common\Mount & Blade II Bannerlord"
$OUTPUT_DIR = "Z:\MyBannerlordMods\AFmod\AF-beta0.1.0"
# ====================================================
```

如需修改路径，用文本编辑器打开 `build.ps1` 修改这两行即可。

## 编译流程

脚本会自动执行以下步骤：

1. **验证配置** - 检查游戏路径和项目文件是否存在
2. **清理旧的编译产物** - 删除 bin 和 obj 文件夹
3. **编译项目** - 使用 dotnet build 编译 Release 版本
4. **准备输出目录** - 创建必要的目录结构
5. **复制DLL文件** - 复制 AnimusForge.dll 和依赖库
6. **复制资源文件** - 复制 GUI、ModuleData、ONNX 等资源
7. **清理源码目录** - 删除源码目录中的 bin 和 obj 文件夹

## 错误提示

脚本包含详细的错误检测和提示：

- **游戏路径错误** - 如果骑砍本体路径不存在，会显示"ERROR: Game root path does not exist!"
- **游戏安装不完整** - 如果找不到 bin\Win64_Shipping_Client 目录，会提示游戏安装不完整
- **项目文件缺失** - 如果找不到 AnimusForge.csproj，会提示项目文件位置错误
- **编译失败** - 显示详细的编译错误信息和常见问题提示
- **文件复制失败** - 如果复制文件时出错，会显示具体的错误信息

## 文件说明

- `一键编译.bat` - 批处理启动器，双击运行
- `build.ps1` - PowerShell 编译脚本（主脚本）
- `build.bat` - 备用批处理脚本（不推荐使用）
- `setup.bat` - 配置向导（已废弃，直接编辑 build.ps1 即可）
- `README.md` - 本说明文档

## 系统要求

- Windows 10/11
- .NET SDK（用于编译）
- PowerShell 5.0 或更高版本

## 故障排除

### 如果编译失败

1. 检查是否安装了 .NET SDK
2. 检查骑砍本体路径是否正确
3. 查看错误信息，确认缺少哪些依赖

### 如果PowerShell脚本无法运行

以管理员身份运行 PowerShell，执行：
```powershell
Set-ExecutionPolicy RemoteSigned
```

或者直接右键 `一键编译.bat` 选择"以管理员身份运行"
